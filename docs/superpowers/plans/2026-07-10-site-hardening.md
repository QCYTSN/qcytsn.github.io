# Portfolio Security, SEO, and Performance Hardening Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add production security headers, restore search indexability, improve sharing metadata, reduce cover-image weight, synchronize document language, and clear the known runtime dependency audit.

**Architecture:** Wrap all Worker responses with one pure header-hardening function, keep search files as static public assets, and extend the existing metadata and language state without changing navigation. Convert existing project covers to WebP and use an exact npm override for patched PostCSS.

**Tech Stack:** React 19, TypeScript, Vinext/Cloudflare Worker, Next metadata, static SEO files, WebP, Node test runner, npm audit.

## Global Constraints

- Preserve the current editorial design, bilingual copy, single-page hash navigation, and all research demos.
- Use `https://peanut-ai.dev` as the only canonical production origin.
- Do not add analytics, forms, third-party scripts, authentication, database bindings, or new external services.
- Keep CSP compatible with React/Vinext hydration and local static assets.
- Retain dependency changes only if the complete production verification passes.
